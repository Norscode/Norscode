# Norscode Package Manager — Fase 4

Status: **Under development (June 2026)**

## Overview

Norscode package manager uses:
- **norcode.toml** — Project manifest (metadata, dependencies, scripts)
- **norcode.lock** — Lock file (exact versions, reproducible builds)
- **Registry API** — Central package registry (HTTP-based)

## norcode.toml Format

Minimal example:

```toml
[project]
name = "my-app"
version = "1.0.0"
authors = ["Jane Doe <jane@example.com>"]
description = "My Norscode application"
edition = "2026"  # Language version

[dependencies]
http = "^1.2.0"
json = "2.0"
logging = { version = "1.*", optional = true }

[dev-dependencies]
test-utils = "0.1"

[features]
default = ["logging"]
full = ["logging"]

[scripts]
build = "nc bygg --mål elf64 src/main.no"
test = "nc test"
fmt = "nc format src/"

## norcode.lock Format

Lock file ensures reproducible builds. Generated by `nc lock`, committed to version control.

```toml
# This file is automatically generated by `nc lock`
# Do not edit manually

[[packages]]
name = "http"
version = "1.2.3"
hash = "sha256:abc123..."
source = "registry"

[[packages]]
name = "json"
version = "2.0.0"
hash = "sha256:def456..."
source = "registry"

[[packages]]
name = "logging"
version = "1.5.2"
hash = "sha256:ghi789..."
source = "registry"
optional = true
